Рекомендации по настройке безопасности

для Linux

1.Обновления.
Один из основных способов обеспечить безопасность операционной системы — обновлять программное обеспечение. Обновления часто исправляют найденные ошибки и критические уязвимости.
Рекомендуется автоматически устанавливать только обновление безопасности.Можно настроить отправку оповещений о новых обновлениях по электронной почте. Также для поддержания безопасности в ядре Linux есть защитные расширения, например, SELinux. Такое расширение поможет сберечь систему от неправильно настроенных или опасных программ. SELinux — это гибкая система принудительного контроля доступа, которая может работать одновременно с избирательной системой контроля доступа. Запущенные программы получают права на доступ к файлам, сокетам и прочим процессам, и SELinux устанавливает ограничения так, чтобы вредные приложения не смогли сломать систему.

2.Пароль пользователя.
Операционная система Linux это многопользовательская система с учетом обеспечения безопасности для всех пользователей. Для повышения надежности парольной защиты необходимо руководствоваться следующими рекомендациями:
-длина пароля должна быть минимум из 8 букв разного регистра, цифр и других специальных знаков;
-установить период устаревания паролей (это можно сделать с помощью утилиты chage);
-ограничить переиспользование старых паролей;
-установить предельное количество неудачных попыток входа пользователя.

3.Настройка прав доступа пользователей.
Пользователи должны иметь доступ, соответствующий их рангу и ответственности. В Linux можно устанавливать права доступа на файлы и директории.Каждый файл принадлежит одному конкретному пользователю. Даже если сервер администрирует один человек, для различных программ создаются несколько аккаунтов.
Помимо отдельных аккаунтов существует категория доступа для групп. Каждый файл принадлежит одной группе. Один пользователь может принадлежать к нескольким группам. Для категорий пользователей есть возможность устанавливать типы доступа. Обычно это права на запуск, чтение и изменение файла. Разрешения тмпов доступа отмечены буквами: r – чтение, w – изменение, x – запуск.

4. Настройка общей памяти.
По умолчанию весь объем общей памяти /run/shm доступен для чтения и записи с возможностью выполнения программ. Это считается брешью в безопасности и многие эксплойты используют /run/shm для атак на запущенные сервисы. Для большинства настольных, а особенно серверных устройств рекомендуется монтировать этот файл в режиме только для чтения.
Однако, некоторые программы могут не работать, если /run/shm доступен только для чтения. В таком случае надо сохранить возможность записи, но запретить выполнение программ.
